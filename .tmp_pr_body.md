## Overview
This PR modernizes the repository structure and universalizes the core engine to improve support for local action usage and public distribution.

## Key Changes
- **Universal Logic**: Updated the entry point to intelligently fallback to `github.actor` if no explicit `monitoredUser` is provided. The engine is no longer hardcoded to specific user defaults.
- **Improved Metadata**: Moved the `action.yml` from a nested subdirectory to the root. This is the standard pattern for GitHub Actions and ensures better compatibility with both local usage and the GitHub Marketplace.
- **Refined Documentation**: Updated the README to reflect the project's new identity as the **Project Board Rule Engine** and generalized configuration examples.
- **Build Infrastructure**: Added a `build` script to `package.json` for easy bundling via `ncc`.

## Rationale
Moving these files and generalized the inputs reduces friction for developers using this tool outside of the primary organization and aligns with GitHub's best practices for Action development.
